01 문제 정의

인디 아티스트와 팬, 두 축 모두에게 필요한 플랫폼

자본 없는 아티스트는 유행을 따라갈 수밖에 없고, 팬들은 자신이 발굴한 아티스트를 효과적으로 알릴 수단이 없습니다. FAN:STAGE는 이 간극을 메웁니다.

타겟 고객

  • 1차 음악 애호가·팬 (20-35세), 큐레이션과 공유에 적극적인 Z·MZ 세대
  • 2차 독창성을 유지하며 팬베이스를 만들고 싶은 인디·신인 아티스트

핵심 페인포인트

  • 마케팅 비용 없이 바이럴을 만들기 어려운 아티스트
  • 팔로워 기반 노출에 갇혀 있는 기존 SNS
  • 알고리즘이 인기곡 위주로 편향된 음악 발견 경험

비즈니스 기회

팬덤 플랫폼 팬딩, 위버스의 성장과 함께 팬 커뮤니티 시장은 이미 검증된 잠재력을 가지고 있습니다. 개성 있는 음악을 원하는 팬과 진정성 있는 노출을 원하는 아티스트의 니즈가 교차합니다.

02 트렌드 & 인사이트

팬이 주도하는 음악 시장의 새로운 파도

디지털 스트리밍의 성장은 팬이 생산자이자 유통자로 활약하는 ‘팬덤 4.0’ 시대로 이어지고 있습니다.

팬덤 4.0

팬은 이제 소비자가 아닌 매개자이자 창작자입니다. 자발적 참여가 새로운 성장을 이끕니다.

커뮤니티 커머스

사용자가 콘텐츠를 만들고 추천할 때 전환율과 체류 시간이 비약적으로 상승합니다.

플랫폼 공백

위버스, 팬딩 등은 성장했지만 인디 아티스트 중심, 팬 큐레이션 기반 플랫폼은 부재합니다.

차별화 기회

  • 위버스의 문턱을 낮추고
  • 팬딩에 음악 특화 기능을 더하고
  • 스포티파이의 추천력과 커뮤니티를 결합합니다.

03 핵심 컨셉 & 기능

팬이 만드는 음악 생태계

SCAMPER 아이디어로 확장한 기능들을 P0-P2 우선순위로 구현합니다.

P0 · 필수 기능

  • 팬 큐레이션 피드 큐레이션이 곧 발견 알고리즘. 좋아요·재생으로 노출이 재조정됩니다.
  • 연관 아티스트 추천 협업 필터링과 장르 기반 추천으로 열린 발견 경험 제공.
  • 아티스트 피드 신곡, 공연, 소식을 직접 팬에게 전하는 공간.
  • 음악 스트리밍 재생 큐, 플레이리스트, 백그라운드 재생까지 완비한 기본기.

P1 · 성장 기능

  • 팬 등급 시스템 씨앗→숲 팬 등급으로 활동을 게임화, 영향력이 높은 큐레이터 발굴.
  • 아티스트 소통 댓글, 피드백, 라이브 Q&A를 통해 팬-아티스트 관계 강화.

P2 · 확장 기능

팬 펀딩과 라이브 인터랙션으로 수익화·커뮤니티 완성도를 높입니다.

핵심 차별화

  • 팬 큐레이션이 곧 추천 알고리즘
  • 연관 아티스트 자동 노출로 열린 피드 구성
  • 팬 활동 데이터를 인사이트로 전환

Elevator Pitch

"FAN:STAGE는 팬이 직접 아티스트를 큐레이팅하고, 그 큐레이션이 다른 팬들에게 자동으로 노출되는 음악 커뮤니티 플랫폼입니다."

04 서비스 구조

앱 전반을 팬 중심으로 재구성

핵심 여정은 홈 피드에서 발견·재생·소통으로 이어지는 자연스러운 루프입니다.

탭 구조

  • 홈 팬 큐레이션 피드, 추천 아티스트, 실시간 인기 큐레이션
  • 발견 연관 아티스트 네트워크, 장르별 신인, 맞춤 추천
  • 플레이어 하단 고정 미니 플레이어와 확장형 재생 경험
  • 아티스트 프로필, 음악, 피드, 팬 큐레이션을 한눈에
  • 마이 큐레이터 등급, 플레이리스트, 좋아요, 설정

주요 화면 하이라이트

  • 팬 큐레이션 카드: 커버, 작성자, 좋아요/재생 수 시각화
  • 발견 화면: 추천 이유를 함께 제시하는 카드 디자인
  • 플레이어: 가사, 공유, 아티스트 이동을 자연스럽게 연결
  • 아티스트 페이지: 팬 큐레이션 모음으로 열린 노출

데이터 모델 (핵심 컬렉션)

  • Users 선호 장르, 큐레이터 등급, 팔로잉 아티스트
  • Playlists 노출 점수, 장르 태그, 공유/재생 데이터
  • Tracks 재생 수, 좋아요, 가사, 메타데이터
  • Interactions 좋아요, 재생, 팔로우 등 행동 로그
  • Artist Posts & Comments 소통을 기록하는 커뮤니티 데이터

06 12주 MVP 로드맵

3단계 빌드 플랜

핵심 기능을 빠르게 검증하며 커뮤니티와 수익화를 향해 나아갑니다.

Phase 1 · Core (Week 1-6)

  • 소셜 로그인 및 프로필 설정
  • 음원 업로드 & 스트리밍 플레이어
  • 팬 큐레이션 피드 게시 & 상호작용
  • 아티스트 페이지 & 팔로우
  • 연관 아티스트 추천 API (V1)

Phase 2 · Community (Week 7-10)

  • 팬 등급 시스템 (씨앗 → 숲)
  • 아티스트 소통 채널 (댓글·피드백)
  • 커뮤니티 활동 데이터 시각화

Phase 3 · Launch (Week 11-12)

  • 검색 및 필터 고도화
  • 맞춤 알림 & 추천 아티스트 알림
  • SNS 공유 기능으로 확산 유도

07 기술 스택 제안

3인 이하 팀을 위한 현실적인 선택

빠른 실행을 위해 Firebase 기반 스택을 추천하며, 선택적으로 Node.js 백엔드로 확장 가능합니다.

Frontend

  • React Native + TypeScript
  • Redux Toolkit & React Navigation
  • React Native Paper UI 라이브러리

Backend

  • Firebase Authentication · Firestore · Storage
  • Cloud Functions & FCM
  • 대안: Node.js, Express, MongoDB, AWS S3

음악 & 추천

  • HLS 스트리밍, react-native-track-player
  • 추천: Python + Scikit-learn, Firebase ML Kit

디자인 & 협업

  • Figma 프로토타입
  • Notion & Slack으로 커뮤니케이션

08 간이 검증 계획

팬과 아티스트의 온도 체크

테스트 가능한 가설을 세우고, 실사용 데이터를 통해 제품-시장 적합성을 검증합니다.

가설 1

팬 큐레이션 피드는 기존 SNS 대비 아티스트 노출을 2배 이상 확대한다.

  • MVP 사용자 50명 확보
  • 큐레이션 노출·재생 전환율 측정
  • 아티스트 설문으로 체감 노출도 평가

가설 2

팬 등급 시스템은 큐레이션 생성량을 30% 이상 증가시킨다.

  • 등급 도입 전후 활동량 비교
  • 상위 큐레이터 인터뷰

가설 3

연관 아티스트 추천은 신규 아티스트 발견율을 40% 향상시킨다.

  • 추천 클릭 → 재생 → 팔로우 전환 추적
  • 추천 이유 A/B 테스트
