음악 팬 커뮤니티 플랫폼

팬이 직접 빛을 비추는
인디 아티스트를 위한 무대

FAN:STAGE는 팬 큐레이션을 발견 알고리즘으로 전환해, 자본 없이도 인디 아티스트가 발견되고 성장할 수 있는 열린 음악 생태계를 만듭니다.

문제 또는 기회 정의

인디 음악 생태계의 단절된 연결고리

자본 없이도 음악이 발견될 수 있는 순환 구조가 필요합니다. 팬과 아티스트 모두에게 답답함만 남긴 기존 플랫폼의 공백을 FAN:STAGE가 메웁니다.

핵심 문제

“진짜 음악 팬들은 자신이 발굴한 아티스트를 확산시킬 수 있는 무대가 없다.”

  • 자본력 중심 노출 구조로 인디 아티스트는 설 자리 부족
  • 팬의 자발적 큐레이션도 닫힌 SNS에 갇혀 확장이 어려움
  • 알고리즘은 인기곡 위주, 새로운 발견의 판이 협소함

타겟 고객

  • 1차: 음악 팬 (20-35) 인디 발굴, 취향 공유, 커뮤니티 활동에 적극적
  • 2차: 인디 아티스트 자본 없이 팬베이스가 필요, 고유한 색 유지 희망

팬과 아티스트 모두 ‘열린 발견’과 ‘실질적 영향력’을 갈망합니다.

비즈니스 기회

  • 팬덤 플랫폼 성장: 팬딩 월 매출 20억원
  • 팬 주도 커뮤니티 커머스의 성공 사례 다수
  • 인디 시장의 성장 잠재력 + 팬 참여 욕구의 결합

기존 플랫폼이 해결하지 못한 ‘팬 주도 발견’ 공백을 FAN:STAGE가 채웁니다.

트렌드 · 시장 · 고객 인사이트

팬덤 4.0 시대, 음악 커뮤니티의 가능성

293억 달러 2025년 글로벌 스트리밍 시장 규모
12.57% 글로벌 음악 시장 CAGR (2023-2028)
50%+ 국내 음악 매출 중 디지털 비중
59.4% 커뮤니티 커머스 주 1회 이상 이용 비율

팬 행동 심층 인사이트

Z세대의 73%는 자신을 “팬”이라고 정의하고, 90%는 주 1회 이상 유튜브에서 팬 콘텐츠를 소비합니다. 팬은 이제 능동적 생산자이며 브랜드를 움직이는 동력입니다.

경쟁사 공백

  • 위버스: 대형 기획사 중심, 인디 진입 장벽
  • 팬딩: 음악 특화 기능 부재
  • 스포티파이: 추천 강점, 커뮤니티 약점
  • 유튜브 & X: 커뮤니티 활발하지만 음악 특화 경험 부족

FAN:STAGE는 팬 큐레이션 × AI 추천을 결합한 최초의 음악 특화 커뮤니티입니다.

팬 여정의 병목

팬들은 여러 플랫폼을 오가며 발견·응원·공유를 반복합니다. 그러나 그 에너지가 아티스트의 실질적 성장으로 이어지지 않습니다. FAN:STAGE는 이 여정을 하나로 연결합니다.

플랫폼 컨셉

팬이 만드는 음악 생태계

SCAMPER 아이디어 확장을 바탕으로, 팬의 큐레이션과 참여 데이터를 플랫폼의 성장 엔진으로 전환합니다.

Fan Discovery Engine

내가 좋아한 아티스트를 기반으로 연관 팬과 새로운 아티스트를 자동 매칭하여 자연스러운 확산을 유도합니다.

큐레이터 등급 시스템

활동량과 반응 데이터를 점수화해 영향력 있는 팬에게 더 큰 노출과 배지를 제공합니다.

팬 릴레이 플레이리스트

좋아하는 아티스트를 통해 자동으로 연결되는 릴레이 구성이 새로운 음악 발굴 흐름을 만듭니다.

팬 펀딩 + 스트리밍

재생과 동시에 소액 후원을 결합해 아티스트의 지속가능한 활동을 지원합니다. (P2)

팬 큐레이션

플레이리스트 생성 시 피드에 자동 게시

피드 반응 데이터

재생수, 좋아요, 공유, 팬 등급 반영

노출 강화

아티스트 성장

팔로워 증가, 팬 소통, 후원까지 연결

서비스 구조

앱 핵심 탭으로 보는 FAN:STAGE 경험

홈

팬 큐레이션 피드와 추천 아티스트를 한눈에. 실시간 인기 큐레이션이 팬의 열기를 증폭시킵니다.

발견

내가 좋아한 아티스트 기반 연관 추천, 장르별 신인 아티스트, 그래프형 네트워크까지. 발견의 즐거움이 배가됩니다.

플레이어

하단 고정 플레이어와 확장 화면에서 가사, 큐레이션, 아티스트 정보를 즉시 확인합니다.

아티스트

프로필, 음악, 피드, 팬 큐레이션이 집약된 아티스트 허브. 팔로우와 소통까지 한 번에.

마이

내가 만든 플레이리스트, 좋아요, 팔로잉, 큐레이터 등급을 관리하며 나만의 활동 히스토리를 확인합니다.

차별화 포인트

팬의 영향력을 극대화하는 세 가지 엔진

팬 큐레이션 기반 노출

팬이 만든 플레이리스트가 곧 추천 지표가 되어, 바이럴 구조를 팬들이 주도합니다.

연관 아티스트 자동 추천

협업 필터링과 장르/연관 그래프를 결합해 열린 피드에서 새로운 아티스트를 발견합니다.

팬 × 아티스트 직접 소통

댓글, 피드백, 라이브 등 다양한 접점으로 팬 참여가 아티스트 성장을 직결합니다.

12주 개발 로드맵

Phase별 우선순위와 기술 스택

Phase 1 · Week 1-6

Core Features

소셜 로그인, 프로필, 음악 스트리밍, 큐레이션 피드, 아티스트 페이지, 연관 추천까지 핵심 경험을 완성합니다.

  • Firebase Authentication · Firestore · Storage
  • React Native (TypeScript), react-native-track-player
  • 협업 필터링 기반 추천 첫 버전
Phase 2 · Week 7-10

Community Features

팬 등급 시스템과 아티스트 소통 기능으로 팬 참여도를 고도화합니다.

  • 큐레이터 포인트 & 랭크, 배지 노출
  • 아티스트 피드 댓글 및 상호작용
Phase 3 · Week 11-12

Polish & Launch

검색, 알림, 공유 기능을 더해 런칭 준비를 마무리합니다.

  • 검색 & 필터, 추천 아티스트 알림
  • 플레이리스트/아티스트 공유 기능

기술 & 데이터

빠른 구현을 위한 현실적인 기술 스택

Frontend · 앱

  • React Native + TypeScript
  • Redux Toolkit, React Navigation
  • Design: Figma, React Native Paper

Backend

  • Firebase: Auth, Firestore, Storage, Cloud Functions, FCM
  • 대안: Node.js + Express, MongoDB, AWS Stack

추천 알고리즘

  • Python (Flask) 기반 API
  • Scikit-learn 협업 필터링
  • Firebase ML Kit 활용

데이터베이스 스키마

Users, Artists, Tracks, Playlists, User Interactions, Artist Posts, Comments 컬렉션으로 팬 행동과 콘텐츠를 정밀 추적합니다.

목표 검증

간이 검증 계획

가설

  • 팬 큐레이션 피드가 음악 발견 경험을 향상시킨다.
  • 팬 등급 시스템은 참여도를 높인다.
  • 연관 추천은 신규 아티스트 청취로 이어진다.

지표

  • 플레이리스트 생성/공유 수
  • 연관 추천 통한 신규 재생 수
  • 팬 등급 상승률, retention

실행 계획

100명의 베타 팬과 30명의 인디 아티스트를 모집해 4주간 사용성 테스트를 진행하고 정량/정성 지표를 수집합니다.
